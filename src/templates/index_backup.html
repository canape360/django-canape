Django Canape プロジェクト

Django Canape の開発・管理・Git運用手順をまとめた HTML版 README です。

1. 環境セットアップ

プロジェクトディレクトリに移動

cd ~/projects/django-canape

仮想環境の起動

source myenv/bin/activate

必要なパッケージをインストール

pip install -r requirements.txt

src ディレクトリへ移動

cd src

2. 開発用サーバー起動

python3 manage.py runserver

開発サーバーはデフォルトで http://127.0.0.1:8000/ でアクセス可能です。

ポートを変更する場合：

python3 manage.py runserver 8001

3. 静的ファイルの管理

python manage.py collectstatic --noinput
  • STATICFILES_DIRS にあるファイルは個別に配置してください。
  • STATIC_ROOT に収集されます。

4. 管理画面でのデータ操作

admin.py にモデルを登録することで、管理画面に表示されます。

Python シェルでのデータ確認・編集

python3 manage.py shell
from myapp.models import MyModel

# データ取得
q = MyModel.objects.get(id=1)

# データ表示
print(q.title)
print(q.content)

# データ変更
q.title = "新しいタイトル"
q.save()

5. Git の操作

保留中の変更確認

git status

変更をステージング

git add .

特定ファイルだけステージングする場合：

git add src/main/settings.py 説明.md

コミット

git commit -m "変更内容の説明"

リモートリポジトリに反映

git push

保留中変更を破棄（削除）したい場合

git restore .
git clean -df

⚠️ 注意：git clean -df は未追跡ファイルを削除します。必ず必要なファイルがコミット済みか確認してください。

7. 注意事項

  • 静的ファイルの配置場所は間違えないようにしてください。
  • データベース（SQLite や PostgreSQL）の変更はバックアップを推奨します。
  • Git の操作前には必ず git status で保留中変更を確認してください。
